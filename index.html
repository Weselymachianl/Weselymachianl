// autobiography

自傳

我是陳俞閔，目前就讀於國立中正大學通訊工程研究所。我是一位具備 AI 演算法開發與系統整合實作經驗的應屆畢業生，擅長從數據中分析邏輯，並透過系統化的思維解決技術瓶頸。

大學就讀於國立高雄科技大學通訊工程學系期間，我對演算法設計與系統實作產生濃厚興趣。在大四專題「QR Code 掃描優化系統」中，利用了 Python 開發核心演算法，成功克服了影像干擾與汙損環境下的辨識難題。該專案不僅整合了跨平台網頁展示，更憑藉解決實際痛點的高實用性榮獲畢業專題佳作。

升上碩士後，我的研究聚焦於 AI 演算法與系統最佳化的應用。面對流體天線系統（FAS）中複雜的參數優化難題，我透過結合神經網路模型與深度強化學習，讓系統能在「資訊不完整、具備多變數」的環境下，自動學習並尋找最佳的參數配置，以最大化系統效能。在系統測試過程中，我設立了客觀且條件一致的模擬環境進行效能評估。實驗結果顯示，我所設計的演算法在各項測試的曲線表現上皆達到最佳狀態，成功突破傳統方法的瓶頸。這段歷程不僅深化了對機器學習框架（PyTorch）的掌握，更培養了我從數據中梳理邏輯、透過反覆實驗排除異常（Debug）的解題能力。

在技術廣度上，我具備將技術落地並解決實際問題的整合能力。例如在開發多人即時通訊系統時，我運用 Node.js 建構後端，並導入 Docker 容器化技術以確保系統在不同開發環境下都能維持一致的穩定度。秉持著「只要肯學就能掌握」的積極態度，我做事嚴謹、條理分明，善於團隊協作。我期待將這份扎實的學術訓練與重視穩定性的實作精神帶入職場，為企業創造實際的工程價值。

// strengths & weaknesses

競爭優勢與需改善點

關於我的競爭優勢，我認為在於「跨域整合的實作能力」與「嚴謹的系統化思維」。
在專業上，我不僅具備開發 AI 演算法的能力，也熟悉 Node.js 與 Docker 等後端與容器化部署技術。這讓我在面對問題時擁有全局觀，能從底層邏輯一路思考到系統上線的穩定性。在個性上，我做事嚴謹且實事求是，在遇到 Bug 或是效能瓶頸時，我習慣將問題拆解，透過反覆的控制變因測試來找出根本原因（Root Cause），並建立防呆機制。

至於需要改善與持續精進的地方，我認為是「對真實工業級數據與實務場景的處理經驗」。
過去在學術研究中，為了確保效能評估的基準公平性，我的模型測試多處於「參數受控的客觀模擬環境」。但我深知，業界實際產線或系統中，往往伴隨著各種未知的雜訊、機台特例與複雜的實務邏輯。
為了解決這個落差，我會發揮自己『只要肯學就能掌握』的積極度。在進入職場後，我會主動向資深前輩請教，並積極研讀系統文件與標準作業流程（SOP）。我相信憑藉著過去訓練出來的系統化解題邏輯，我絕對能快速跨越學習曲線，將理論知識轉化為解決實務問題的即戰力。

// failures & achievements

挫折與成就感

最感挫折的經驗與克服過程

我求學過程中最感挫折的經驗，是在碩士研究初期開發 AI 演算法的階段。當時我為了找出最佳參數，一開始採用了純粹的決策型深度強化學習（DRL）。但我很快就遇到了嚴重的效能瓶頸：我發現基礎模型「缺乏記憶性」，導致訓練過程不斷在原地打轉。

遇到這個挫折後，我查閱了大量文獻，將架構升級為「具備記憶性的深度強化學習」。然而解決了記憶問題後，模型卻又陷入了「局部最佳解」的陷阱。

面對這種連環卡關，我再次深入研究，最終決定導入「預測型的神經網路模型（如 ConvCNP）」來輔助決策。在評估這個解法時，我特別把「計算複雜度」納入考量。因為當初導入深度強化學習的初衷，就是要降低系統整體的運算負擔，如果為了追求最佳解而疊加過於龐大、耗時的架構，那就是本末倒置了。而挑選出的神經網路模型，恰好能在不大幅增加運算複雜度的前提下，成功引導系統跳出局部盲區，讓模型順利且高效地收斂。

最有成就感的經驗

而這段充滿挫折的過程，剛好也是我求學階段最有成就感的經歷。

對我來說，最有成就感的那一刻，並不是一開始就寫出完美無缺的程式，而是當我經過無數次的「遇到瓶頸、尋找文獻、評估運算成本、修改底層架構」後，看著最終的測試結果不僅效能大幅改善，更完美守住了「低運算複雜度」的研究初衷。

這次的經驗讓我深刻體會到，做研究或開發系統一定會遇到理論與實務的落差。但我具備了「能獨立尋找解法，並在效能與成本之間做出最佳工程權衡（Trade-off）」的能力，這份突破困難後結出的果實，是我最有成就感的來源。
